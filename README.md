# 项目

本项目是通过 `libpcap` 这个库，实现局域网网络抓包，通过抓包分析，来统计网络流量，分析流量并能判断小孩是否在玩网络游戏或上网。

本项目的目的是让学员学会使用第三方库抓取网络数据包，并分析数据包，达到网络监控的目的。同时能够综合运用已学过的 C 语言知识来完成较复杂的需求。



### 要求

>要求以小时为单位，统计家里每一个 `IP` 地址的外网流量，将统计报告写入文件当中，文件名类似 `2024-09-01T23:00`。打开这个文件就知道家里每个 `IP` 访问外网情况。

- 首先，学员需要自行安装 `libpcap` 这个库，并知道怎样把这个库链接到自己的项目中来进行使用。

- 学员需要对 `TCP/IP` 协议的数据包格式非常熟悉，需要根据包格式定义数据结构。
- 学员需要掌握 `libpcap` 抓包的基本流程。
- 尝试学会用责任链的设计模式来实现解耦^[责任链介绍请看文档最后]。

当 `libpcap` 捕获到数据包的时候，先进行以太网帧的解析，可以得到源 `MAC` 和目的 `MAC` 地址。然后把以太网帧的数据（包含 `IP` 层数据）丢给下一个节点去处理，下一个节点解析出 `IP` 头，把数据在丢给下个节点…

我们可以用树这种数据结构来表示这样一个职责链，然后按层序遍历即可处理完数据包。

学员需要掌握网络协议栈中数据包的格式，并定义相应的数据结构，要注意正确的应用位域来优化结构体大小，例如以下的IP头部，需要知道每个字段的大小。

### `libcpeg`库抓包流程

1. 查找设备(查询网卡设备)

```c
// 可用的的设备
int ret = pcap_findalldevs(&devs, errbuf);
if (ret == -1) {
    printf("no dev up err gsn", errbuf);
    return 0;
}
```

2. 打开设备并开始抓包

```c
// 打开设备
handle = pcap_open_live(dev->name,BUFSIZ, 1, 1000, errbuf);
// 传递packet_count统计包个数
pcap_loop(handle, -1, handler, (unsigned char *)&packet_count);
```

3. 处理数据(抓包回调函数`handler`)

```c
handler(u_char *args, const struct pcap_pkthdr *hdr, const u_char *packet);
```

第二个参数包含抓到的数据包的元数据信息，如时间戳，包长度等，第三个参数数据包实际内容的指针。我们就是需要解析这个数据包。

### 责任链

责任链模式是一种行为设计模式，它允许你将请求沿着处理者链进行传递，直到有一个处理者能够处理该请求为止。这种模式将请求的发送者和接收者解耦，使得多个对象都有机会处理请求。

#### 传统责任链模式

在传统的责任链模式中，处理者通常形成一条线性链。每个处理者都包含对下一个处理者的引用，当一个处理者收到请求时，它会先判断自己是否能够处理该请求，如果可以则处理，否则将请求传递给下一个处理者。

#### 树状责任链模式

树状责任链模式是责任链模式的一种扩展，它使用树结构（通常用孩子兄弟表示法实现）来组织处理者。这种模式允许请求在多个方向上传递，形成更复杂的处理网络。
- 孩子兄弟表示法
    - 孩子兄弟表示法是一种将多叉树转换为二叉树的方法，每个节点包含两个指针：
    - 第一个孩子指针：指向该节点的第一个子节点
    - 下一个兄弟指针：指向该节点的下一个兄弟节点

通过这种方式，可以将复杂的树状结构简化为二叉树，便于实现和遍历。

#### 责任链模式与树结构结合的优势

- 解耦请求发送者和接收者：请求的发送者不需要知道具体是哪个处理者处理了请求，只需要将请求发送到链的起点即可。
- 动态组合处理者：可以在运行时动态地添加、删除或修改处理者，灵活地调整责任链的结构。
- 支持复杂的处理路径：树状结构允许请求在多个处理者之间分支和传递，形成更复杂的处理网络。
- 可扩展性：可以轻松地添加新的处理者类，而不需要修改现有的代码，符合开闭原则。
